Для добавления изображений лучше всего использовать картинки формата векторной графики SVG, чтобы она одинаково хорошо изображалась на всех устройствах.
Мы можем также изменять высоту объектов с помощью атрибута android:layout_weight="значениеdp". По сути он позволяет расположить элементы в каких - то пропорциях. Для плохо видящих или слепых людей для
озвучивания того, что находится на картинке используют атрибут в xml android:contentDescription="текст". После этого они услышат название. Поведение экшн бара и клавиатуры можно настроить в манифесте.
Вот что надо добавить для корректной работы клавиатуры в манифесте: android:windowSoftInputMode="adjustResize". Для того, чтобы экран можно было скроллить используют viewgroup  ScrollView. При этом он может
содержать только один какой - то элемент. Поэтому в данном приложении в третьем макете скролл вью является корневым для остальных элементов, в нем и прописываем зависимости. Также для безопасности используют
метод для строк trim, он вырезает все пробелы перед сообщением и после него. Для отображения ворнингов или уведомлений используется класс Toast. Вызов Toast имеет следующую конструкцию:

Toast.makeText(Класс_где_вызываем_его, "текст_сообщения", Toast.LENGTH_LONG/Toast.LENGTH_SHORT).toShow();
